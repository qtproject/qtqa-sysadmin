RTA HISTORY DESCRIPTION:

Level 1: Show list of test jobs

GENERAL
Purpose Provide visibility to the Release Test Automation (RTA) status and its progress at one glance
Question What is the number of passes and failures (types: ERROR, FAIL, FATAL and XPASS i.e. uneXpected PASS) in each test job, both on the latest and previous test runs.
What are the changes between the latest and previous test run in terms of number of passes and failures.
 
METRIC
Filters Test type, License type, Platform can be used to filter the list.
The test job name links in the list can be used to dive in into detailed information (level 2 view).
Indicator List of test jobs with their status, result history and changes
Calculation The changes are compared only between the latest (last) and previous (last-1) test run, older ones are shown just for reference.
The failure count is a summary of the following types: ERROR, FAIL, FATAL and XPASS i.e. uneXpected PASS. Each failure type is compared separately for the change counts. Therefore it is possible to identify the cases where failures are decreasing with one type, while increasing with some other type.
See the link in "status & changes" for further details.
Interpretation Status & changes (Passes): Numbers indicate the change between the latest (last) and previous (last-1) test run. Color coding is not used.
Status & changes (Failures): Red color indicates the failure count is increasing, yellow indicates there are failures existing, and green indicates that there are not any failures. Numbers indicate the change between the latest (last) and previous (last-1) RTA test job run.
See the link in "status & changes" for further details.
Target value Status & changes: All tests are passed and there are not any failures i.e. the failures show green.
 
DATA
Data items Test job name: Job name in Packaging Jenkins, e.g. "Qt5_RTA_opensource_installer_tests_linux_32bit", where "opensource" is a License type, "installer" is a Test type and "linux" is a Platform (as used in the filters).
Configuration: One or several per test job (e.g. "linux-g++-Ubuntu11.10-x86").
Installer build: Number of the offline/online Qt installer under test.
Status & changes: See the link in "status & changes" for further details.
last: Number of passes and failures in the latest test run (Packaging Jenkins build history number), used in comparison.
last-1: Number of passes and failures in the previous test run, used in comparison. The value "-" indicates that the specific configuration in the related test job run was not executed, or that the results are otherwise not available.
last-n: Number of passes and failures in the previous test runs. See above for the value "-".
Data source Data in result XML files (summary.xml, and Installer build number from result_***.xml) saved from Packaging Jenkins to ci-files02-hki.ci.local/home/bld.qt/squishlogs/.
Files are located under each test job directory (e.g. "Qt5_RTA_opensource_installer_tests_linux_32bit") and their Packaging Jenkins build history number directory (e.g. "220") as configuration specific tar.gz files (e.g. "linux-g++-Ubuntu11.10-x86.tar.gz").
Data update cycle New test job result files are saved in real time
