RTA HISTORY DESCRIPTION:

Level 2: Show test job history and comparison

GENERAL
Purpose Provide list of changes in the results of the last two Release Test Automation (RTA) test job runs to be able to easily analyze those changes and to start needed corrective actions.
Question What are the failures (types: ERROR, FAIL, FATAL and XPASS i.e. uneXpected PASS) both on the latest and previous test runs.
What are the changes between the latest and previous test run failure by failure.
 
METRIC
Filters The selected test job and its configuration is shown
Indicator Test job result history and the changes between the last two test runs
Calculation Test history: The changes are compared only between the latest (last) and previous (last-1) test run, older ones are shown just for reference. See the link in "status & changes" for further details.
Comparison: The summary tables calculate the number of PASSes and each failure type, as well as the number of changes between the last two test runs, on count basis. The failures in the latest (last) and the previous (last-1) test run are compared as text. The summary of changes table counts the failure messages based on are they new, removed or the same in the last two test job runs.
Interpretation Test history: Numbers and the color in the Status & changes indicate the change between the latest (last) and previous (last-1) test run. See the link for further details.
Comparison: The summary tables show the number of PASSes and each failure type, as well as the changes between the last two test runs, on count basis. The failures are listed parallel for the latest (last) and the previous (last-1) test run to identify are they new (appear only in the 'last' run column), are they the same (appear in both columns), or have they been removed (appear only in the 'last-1' column). The summary of changes table show the number of new, removed and same failures in the two runs.
Target value Test history (Status & changes): All tests are passed and there are not any failures i.e. the failures show green.
Comparison: There are not any failures with the latest (last) test job run i.e. the 'last' column show either zero (for failure counts) or empty (for failure descriptions).
 
DATA
Data items Test job name: Job name in Packaging Jenkins, e.g. "Qt5_RTA_opensource_installer_tests_linux_32bit", where "opensource" is a License type, "installer" is a Test type and "linux" is a Platform (as used in the filters).
Configuration: One or several per test job (e.g. "linux-g++-Ubuntu11.10-x86").
Installer build number: Number of the offline/online Qt installer under test.
Jenkins build history: Running number of test jobs in Packaging Jenkins. Note:The squish report link will work only if the report is available in Jenkins.
Status & changes: See the link in "status & changes" for further details.
last: Number of passes, failures and the failure description (failure type and test case name, python script name and line number, and failure description text) in the latest test run (Packaging Jenkins build history number)
last-1: Number of passes, failures and the failure description in the previous test run. Empty or the value "-" indicates that the specific configuration in the related test job run was not executed, or that the results are otherwise not available.
Data source Data in result XML files (summary.xml and result_***.xml) saved from Packaging Jenkins to ci-files02-hki.ci.local/home/bld.qt/squishlogs/.
Files are located under each test job directory (e.g. "Qt5_RTA_opensource_installer_tests_linux_32bit") and their Packaging Jenkins build history number directory (e.g. "220") as configuration specific tar.gz files (e.g. "linux-g++-Ubuntu11.10-x86.tar.gz").
Data update cycle New test job result files are saved in real time
